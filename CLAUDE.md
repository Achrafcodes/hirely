Read DESIGN_SYSTEM.md before touching any component, style, or config file.

After completing any task that changes a design decision — colors, fonts,
spacing, components, tokens — update DESIGN_SYSTEM.md to reflect the
current state of the project. The file should always match what's actually
in the codebase, not what we planned.

When updating DESIGN_SYSTEM.md:
- Change values that were modified
- Add new components or patterns that were built
- Remove decisions that were scrapped
- Never add aspirational notes — only document what exists right now
